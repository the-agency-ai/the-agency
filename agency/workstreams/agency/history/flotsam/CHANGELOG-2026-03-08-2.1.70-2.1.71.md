## Major Features Overview

### 1. Cron Scheduling System (v2.1.71) — NEW CAPABILITY

A full cron scheduling system within Claude Code sessions. This is a significant new capability.

- **`/loop` command** — Run a prompt or slash command on a recurring interval (e.g., `/loop 5m check the deploy`)
- **Three new tools**: `CronCreate`, `CronDelete`, `CronList` — schedule recurring prompts using standard 5-field cron expressions in local timezone
- Session-scoped only (jobs die when session ends)
- Max 50 scheduled jobs per session
- Supports both recurring and one-shot schedules

**Agency relevance:** This could enable automated monitoring patterns — periodic test runs, deployment checks, or status polls without manual triggering. Worth exploring for long-running captain sessions.

### 2. VS Code Integration Overhaul (v2.1.70)

Three significant VS Code additions:

- **Session manager** — Spark icon in activity bar listing all Claude Code sessions with full editor support
- **Plan view** — Full markdown document view for plans with commenting support
- **MCP management** — Native MCP server management dialog accessible via `/mcp` in chat panel

### 3. Token Saver System (v2.1.70) — INTERNAL

New internal mechanism for compressing tool output sent to the model while keeping full output visible in the UI. New SDK fields:

- `tokenSaverOutput?: string` on Bash tool results — compressed output sent to model when token-saver is active
- Internal counters: `tokenSaverBytesSaved`, `tokenSaverHits`

**Agency relevance:** Our Tool Output Standard already minimizes context usage. The token saver works at a different layer (Bash tool output compression). These complement each other — our standard controls what tools emit, the token saver compresses what the model sees from raw bash output.

### 4. Ripgrep Architecture Change (v2.1.71) — INFRASTRUCTURE

Dropped all `ripgrep.node` native Node bindings (6 files removed). Ripgrep now invoked as subprocess (`rg` binary) rather than through Node native bindings. Package file count reduced from 22 to 16.

### 5. Expanded Bash Auto-Approval (v2.1.71)

Added to allowlist: `fmt`, `comm`, `cmp`, `numfmt`, `expr`, `test`, `printf`, `getconf`, `seq`, `tsort`, `pr`.

These are standard Unix text processing utilities — fewer permission prompts for common operations.

### 6. Rebindable Push-to-Talk (v2.1.71)

`voice:pushToTalk` keybinding now configurable via `keybindings.json`.
