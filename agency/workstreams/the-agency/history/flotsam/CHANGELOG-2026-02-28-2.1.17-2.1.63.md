## Major Features Overview

### 1. Claude Opus 4.6 Model Support (v2.1.45+)

The flagship model upgrade bringing improved coding capabilities.

- **1M token context window** available (disable with `CLAUDE_CODE_DISABLE_1M_CONTEXT=1`)
- Fast mode for Opus 4.6 includes full 1M context window
- Model selection improved with human-readable labels via `/model` picker
- Top Terminal-Bench 2.0 scores

**How to use:**
```bash
# Select model interactively
/model

# Or set in environment
export ANTHROPIC_MODEL=claude-opus-4-6
```

### 2. Agent Teams / Multi-Agent Coordination (v2.1.40+, Research Preview)

Teams of Claude Code agents working in parallel on the same codebase.

- Lead agent coordinates work, assigning tasks to teammates
- Teammates work independently in isolated context windows
- Direct team member communication (not just through lead)
- Each teammate gets its own git worktree for isolation

**How to enable:**
```bash
export CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1
```

**Known limitations:** Session resumption, task coordination edge cases, shutdown behavior.

### 3. Automatic Memory System (v2.1.59+)

Claude automatically saves useful context that persists across sessions.

- Remembers project patterns, key commands, user preferences
- Stored as structured markdown in `~/.claude/projects/<project-hash>/memory/`
- Project configs & auto memory shared across git worktrees (v2.1.63)

**How to manage:**
```bash
/memory  # Opens file selector for auto-memory and CLAUDE.md files
```

### 4. Task Management System (v2.1.50+)

Persistent task tracking across multiple sessions with dependency management.

- **TaskCreate** - Create tasks with subject, description, activeForm
- **TaskList** - View all tasks with status (pending/in_progress/completed)
- **TaskGet** - Retrieve full task details including dependencies
- **TaskUpdate** - Update status, add dependencies (addBlockedBy/addBlocks)
- Filesystem-backed: `~/.claude/tasks/<task-list-id>/tasks.json`
- Multi-session coordination via environment variables

**Note:** Task tools currently bypass PreToolUse/PostToolUse hooks (known issue #20243).

### 5. Plan Mode (v2.1.0+, Enhanced Throughout)

Separates research/planning from code execution.

- Read-only operations for code analysis without making changes
- Activate via **Shift+Tab** (twice) or `/plan` command
- **EnterPlanMode** tool - programmatic entry into plan mode
- **ExitPlanMode** tool - presents plan for user approval
- Plan subagent for dedicated planning workflow

**How to use:**
```
Shift+Tab Shift+Tab  # Toggle plan mode
/plan                 # Enter plan mode
```

### 6. Worktree Support & Isolation (v2.1.49+)

Isolated git worktrees for parallel development sessions.

- `claude --worktree feature-auth` creates `.claude/worktrees/feature-auth/`
- Branch: `worktree-feature-auth` based on HEAD
- Multiple parallel sessions without code edit conflicts
- Agent definitions support `isolation: worktree`
- WorktreeCreate/WorktreeRemove hook events (v2.1.50)
- Non-git VCS support via hooks (SVN, Perforce, Mercurial)
- Auto-cleanup: no changes = removed; changes = prompt to keep/remove

**How to use:**
```bash
claude --worktree my-feature
claude -w my-feature  # Short form

# Add to .gitignore
echo ".claude/worktrees/" >> .gitignore
```

### 7. Customizable Keybindings (v2.1.35+)

Full keybinding customization via JSON configuration.

- Edit `~/.claude/keybindings.json` or run `/keybindings`
- Auto-reload on file changes (no restart needed)
- Namespace:action format (e.g., `chat:submit`, `app:toggleTodos`)
- Validation with warnings via `/doctor`
- Terminal multiplexer conflict detection

**How to configure:**
```json
// ~/.claude/keybindings.json
[
  { "key": "ctrl+e", "action": "chat:openExternalEditor" },
  { "key": "ctrl+enter", "action": "chat:submit" }
]
```

### 8. PDF Page Ranges (v2.1.30+)

Direct PDF reading with page-range support.

- `pages` parameter: `"1-5"`, `"3"`, `"10-20"` (max 20 pages per request)
- Large PDFs (>10 pages) require page parameter
- No conversion needed - native filesystem reading

### 9. Background Agents (v2.1.49+)

Agents that run as background tasks.

- Agent frontmatter: `background: true`
- **Ctrl+F** kills background agents (two-press confirmation)
- Progress indicator with accurate tool count

**Agent definition example:**
```yaml
---
background: true
isolation: worktree
model: sonnet
---
# My Background Agent
...
```

### 10. `/debug` Command

Session troubleshooting diagnostics.

- Tool call failures and denials
- Token count, tool uses, duration metrics
- Enhanced git integration

### 11. `--from-pr` Flag

Resume sessions linked to specific GitHub pull requests.

```bash
claude --from-pr 123
```
