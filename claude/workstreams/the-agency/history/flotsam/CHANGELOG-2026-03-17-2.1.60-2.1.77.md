## Major Features Overview

### 1. Output Token Limit Increase (v2.1.77) — SIGNIFICANT

Opus 4.6 default max output raised to **64k tokens**, with upper bound of **128k** for both Opus 4.6 and Sonnet 4.6. This is a major capacity increase for complex code generation.

### 2. Agent Tool Rework — SendMessage Pattern (v2.1.77)

The Agent tool no longer accepts a `resume` parameter. Instead, use `SendMessage({to: agentId})` to continue a previously spawned agent. `SendMessage` auto-resumes stopped agents in the background.

**Agency relevance:** This changes how our multi-agent review workflow communicates with subagents. We need to verify our review-spawn and agent coordination patterns work with SendMessage instead of resume.

### 3. MCP Elicitation (v2.1.76) — NEW CAPABILITY

MCP servers can now request structured input mid-task via interactive dialogs (form fields or browser URL). New `Elicitation` and `ElicitationResult` hooks to intercept/override responses.

**Agency relevance:** If we build custom MCP servers, they can now ask the user questions mid-execution rather than requiring all input upfront.

### 4. 1M Context Window for Opus 4.6 (v2.1.75)

1M context now available by default for Max, Team, and Enterprise (previously required extra usage).

### 5. Cron Scheduling System (v2.1.71-2.1.72)

- `/loop` command — recurring prompt/command execution
- `CronCreate`/`CronDelete`/`CronList` tools — standard 5-field cron expressions
- `CLAUDE_CODE_DISABLE_CRON` env var to stop scheduled jobs (v2.1.72)
- Session-scoped (dies when session ends), max 50 jobs

### 6. Model Overrides (v2.1.73)

`modelOverrides` setting maps model picker entries to custom provider model IDs (e.g., Bedrock inference profile ARNs). Useful for custom deployments.

### 7. Worktree Sparse Checkout (v2.1.76)

`worktree.sparsePaths` setting for `claude --worktree` in large monorepos — only checks out the directories you specify via git sparse-checkout.

### 8. `/plan` with Description (v2.1.72)

`/plan fix the auth bug` enters plan mode and immediately starts planning with that description.

### 9. Effort Level Simplification (v2.1.72)

Effort simplified to **low/medium/high** (removed "max"). New symbols: ○ ◐ ●. `/effort auto` resets to default. `/effort` slash command added in v2.1.76.

### 10. `/context` Command Enhanced (v2.1.74)

Now shows actionable suggestions — identifies context-heavy tools, memory bloat, capacity warnings with specific optimization tips.

### 11. `/copy` Enhancements (v2.1.60-2.1.77)

- Interactive code block picker (v2.1.60)
- "Always copy full response" option (v2.1.69)
- `/copy N` copies Nth-latest response (v2.1.77)
- `w` key writes selection directly to file, bypassing clipboard (v2.1.72)

### 12. `/branch` (formerly `/fork`) (v2.1.77)

`/fork` renamed to `/branch`. `/fork` still works as alias.

### 13. Auto-Memory (v2.1.60+)

Claude automatically saves useful context. `autoMemoryDirectory` setting added in v2.1.74 for custom storage location. Last-modified timestamps added in v2.1.75 for freshness reasoning.

### 14. PostCompact Hook (v2.1.76)

New hook fires after compaction completes. Useful for post-compaction cleanup or context restoration.

### 15. CLAUDE.md HTML Comments Hidden (v2.1.72)

`<!-- ... -->` comments in CLAUDE.md are now hidden from Claude when auto-injected. Still visible when read with the Read tool. Useful for human-only notes in CLAUDE.md.
