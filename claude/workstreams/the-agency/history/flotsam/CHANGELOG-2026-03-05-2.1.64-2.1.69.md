## Major Features Overview

### 1. Effort Level System Changes (v2.1.68)

Opus 4.6 now defaults to **medium effort** for Max and Team subscribers (was: high effort). The "ultrathink" keyword has been re-introduced to enable high effort for the next turn.

- Effort level now displayed in logo and spinner (e.g., "with low effort") (v2.1.69)
- Opus 4 and 4.1 removed from first-party API; auto-migrated to Opus 4.6

**How to use:**
```
# Type "ultrathink" in your prompt to get high effort for that turn
ultrathink analyze this complex codebase

# Or set effort level explicitly via /model
```

### 2. Voice STT Language Expansion (v2.1.69)

10 new languages added (20 total): Russian, Polish, Turkish, Dutch, Ukrainian, Greek, Czech, Danish, Swedish, Norwegian.

### 3. Plugin System Improvements (v2.1.64-2.1.69)

- `/reload-plugins` command to activate pending plugin changes without restart (v2.1.69)
- Plugin search functionality added to installed plugins list (v2.1.64)
- Pin plugins to specific git commit SHAs (v2.1.64)
- `git-subdir` plugin source type to point to subdirectories within git repos (v2.1.69)
- `pluginTrustMessage` managed setting for organization context (v2.1.69)
- `pathPattern` in `strictKnownMarketplaces` for regex-matching git sources (v2.1.69)

### 4. `/claude-api` Skill (v2.1.69)

New bundled skill for building applications with Claude API and Anthropic SDK. Supports both Node.js/TypeScript and Python.

### 5. `InstructionsLoaded` Hook Event (v2.1.69)

New hook event that fires when CLAUDE.md or `.claude/rules/*.md` files are loaded.

### 6. `${CLAUDE_SKILL_DIR}` Variable (v2.1.69)

Skills can now reference their own directory using `${CLAUDE_SKILL_DIR}`, enabling portable skill definitions.

### 7. Bash History Autocomplete (v2.1.64)

History-based autocomplete in bash mode. Type partial command + Tab to complete from bash history.
