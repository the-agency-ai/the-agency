# BUG-0001: iTerm tab title doesn't show status shapes

**Status:** Open
**Reported:** 2026-01-20
**Assigned To:** captain
**Workstream:** housekeeping
**Related:** REQUEST-jordan-0048

## Description

iTerm tab colors work correctly, but status shapes (●, ◐, ▲) don't appear in the tab title. Shapes only show in the badge overlay (visible in the terminal window, not the tab bar).

## Current Behavior

- Tab color changes correctly (blue/green/red)
- Badge shows shape indicator (◐ for working, etc.)
- Tab title shows only the terminal name, no shape

## Expected Behavior

Tab title should show the shape prefix, e.g., "◐ housekeeping/captain"

## Investigation

See `claude/docs/investigations/ITERM-TAB-SHAPES.md` for detailed findings.

**Root cause:** iTerm's tab title is controlled by the shell/terminal escape sequences, but Claude Code runs in a subprocess without direct /dev/tty access. User variables are set but the Tab Title field in iTerm profile needs to reference them.

## Workaround

Badge overlay shows shapes - this is functional but less visible than tab title would be.

## Potential Solutions

1. Configure iTerm Tab Title field to use `\(user.agentIndicator) \(user.agentTitle)`
2. Find a way to set tab title from within Claude Code hooks
3. Accept badge-only solution as sufficient
