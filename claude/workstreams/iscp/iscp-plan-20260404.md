Build agent-identity (prerequisite), then dispatch v2 (highest priority), then flag v2.

### Iteration 1.1: PVR + A&D ✓

**Completed:** Design phase. PVR and A&D with QG.

### Iteration 1.2: `_iscp-db` library ✓

**Completed:** Shared SQLite library. 51 BATS tests. Schema, parameterized queries, convenience helpers.

**Commit:** `9956644`

### Iteration 1.3: `agent-identity` tool [S]

New tool at `claude/tools/agent-identity`. Unified "who am I" with branch-scoped caching.

- Source `_address-parse`, `_path-resolve`, `_log-helper`
- Resolution: branch detection (main/master → captain, worktree → agent slug) + `address_resolve()`
- Cache key includes branch: `~/.agency/{repo}/.agent-identity-{branch-hash}` (MAR: prevent worktree collision)
- Support `CLAUDE_AGENT_NAME` env var override (testing)
- Output: `repo/principal/agent` (default), `--agent`, `--principal`, `--json`
- **Tests:** `tests/tools/agent-identity.bats` (~10 tests)

### Iteration 1.4: `dispatch create` subcommand [M]

Merge dispatch-create INTO dispatch as `create` subcommand (per A&D spec). Thin wrapper for backward compat.

- Source `_iscp-db`, call `iscp_db_init`
- `--type <type>` flag (default: `dispatch`, validated against 8-type enum)
- `agent-identity` for `from_agent` (auto-computed, never self-asserted)
- After creating payload file, INSERT into dispatches table
- `--reply-to` accepts dispatch ID → sets `in_reply_to` FK
- `dispatch-create` becomes thin wrapper: `exec dispatch create "$@"`
- **Tests:** Rewrite `tests/tools/dispatch-create.bats` (~15 tests)

### Iteration 1.5: `dispatch` lifecycle [M]

Complete dispatch tool with DB-backed lifecycle subcommands.

- `list [--all] [--status <s>] [--type <t>]` — DB query, formatted table
- `read <id>` — show payload (from main checkout), mark read in DB
- `check` — `iscp_db_count_unread`, silent when zero, JSON systemMessage when non-zero
- `resolve <id> [--response <id>]` — mark resolved, optionally link response
- `status <id>` — full record
- **Skill updates:** `.claude/skills/dispatch/SKILL.md`, `.claude/skills/dispatch-read/SKILL.md`, `.claude/skills/session-resume/SKILL.md`
- **Tests:** New `tests/tools/dispatch.bats` (~15 tests)

### Iteration 1.6: Flag v2 [S]

Rewrite `claude/tools/flag`. JSONL → SQLite, agent-addressable.

- `flag <message>` — default to_agent = self (per PVR/A&D)
- `flag --to <agent> <message>` — route to specific agent
- `flag list` — mark read on view (Slack-style)
- `flag count`, `flag discuss`, `flag clear`
- Drop jq dependency, drop git-add
- **Skill update:** `.claude/skills/flag/SKILL.md`
- **Tests:** `tests/tools/flag.bats` (~12 tests)

**Phase 1 exit criteria:** Dispatch lifecycle works end-to-end (create, list, read, resolve). Flag tool works from any worktree. Agent identity resolves correctly with branch-scoped cache. Skills updated. ~52 new tests.

## Phase 2: Hook + Migration + Enforcement

### Iteration 2.1: `iscp-check` + hook wiring [S-M]

New tool at `claude/tools/iscp-check` + `.claude/settings.json` updates.

- **Tests:** `tests/tools/iscp-check.bats` (~10 tests)

### Iteration 2.2: Migration + hookify rules [M]

`claude/tools/iscp-migrate` — one-shot migration for flags and dispatches.

- Flag migration: parse JSONL, insert, mark read, rename to `.migrated`
- Dispatch migration: scan `dispatches/` AND `code-reviews/` (MAR finding), parse YAML frontmatter with markdown header fallback, default type=`dispatch` for missing/invalid, map statuses
- Idempotent via UNIQUE index on payload_path

Hookify rules:
- `hookify.dispatch-manual.md`, `hookify.flag-manual.md`
- `hookify.directive-authority.md`, `hookify.review-authority.md`
- `hookify.session-start-mail.md`
- **Tests:** `tests/tools/iscp-migrate.bats` (~10 tests)

**Phase 2 exit criteria:** Agents auto-notified on session start and every turn. Legacy data migrated. Enforcement triangle complete. ~20 new tests.

## Quality Gate Reports

### Iteration 1.2: `_iscp-db` library (2026-04-05)

**Commit:** `9956644` — Phase 1.2: _iscp-db library — shared SQLite abstraction for ISCP

**QG Summary:** 12 findings from 4 agents + own review, all fixed. Key fixes:
- Newline injection in `.param set` values — security vulnerability in core param handling
- Strict param name validation (`^:[a-zA-Z_][a-zA-Z0-9_]*$`) — prevents dot-command injection
- Odd parameter count now rejected (was silently dropped)
- DB file created with `umask 077` (was using default umask)
- 13 new tests added (38 → 51 total)

**QGR receipt:** `usr/jordan/iscp/qgr-iteration-complete-1-2-e0c8972-20260405-0118.md`

**A&D deviation:** `subscriptions.filter` changed from nullable `TEXT` to `TEXT NOT NULL DEFAULT ''` — SQLite treats NULLs as distinct in UNIQUE indexes, which would allow duplicate subscriptions.
