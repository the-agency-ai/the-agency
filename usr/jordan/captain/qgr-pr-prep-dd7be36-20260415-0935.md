## What Was Found and Fixed (narrative)

PR #87 fixed three real bugs in the collaboration tool — incoming cross-repo
dispatches arrive without a status field, dirty trees from prior resolves
broke `git pull`, and `_update_frontmatter_status` silently no-op'd if no
status existed. The fixes were correct in spirit but introduced six edge-case
bugs.

The hardest was the BSD-sed `i\` + `}` construct used to insert a status
line before the closing `---`. On macOS, BSD sed treats the closing `}` as
part of the inserted text or errors out depending on quoting; the result is
unpredictable. Replaced with an awk pass that tracks frontmatter boundaries
explicitly (open at first `^---$`, close at second) and atomically rewrites
via temp file.

The scoping bug (matching `status:` anywhere in the file, including bodies
that quote other dispatch headers) was equally important — replies routinely
quote frontmatter, and the old logic would falsely mark legitimately-unread
dispatches as read. The new awk-based scan exits as soon as the second
`---` is seen.

The stash detection was the subtlest: `git stash` returns 0 even when it
creates no new entry (no tracked changes). The old fix would then `git stash
pop` an unrelated older stash. Now we check `git stash list | wc -l` before
and after.

Six BATS tests cover the awk-based frontmatter rewriter and the new
unread-detection logic. First BATS coverage for the collaboration tool —
broader subcommand coverage flagged as a follow-up.
