## Summary

### Seed-to-Implementation Workflow

- **Seed Generation & Assignment**
  - Seeds created from conversations, ideas, or other inputs
  - Assigned to existing agent on existing workstream OR new workstream with new/existing agent
  - Most likely flow: new workstream + newly created agent

- **PBR Development Process**
  - Seed triggers discussion → produces PBR (Product Problem Vision Requirements Document)
  - Multi-agent research phase before PBR creation (competitor analysis, implementation research)
  - Agent interacts with seed + research input to create PBR
  - All artifacts go through multi-agent review (not code review - focused on viability, competitive mapping)

### Multi-Agent Review Loop Structure

- **Review Type Selection**: Design review, PBR review, A&D review - each has different flow/approach
- **Standard Process**: Artifact → MAR → Result → Disposition
- **Three-Bucket Feedback System**:
  1. Disagreements (presented to principal with reasoning)
  2. Autonomous incorporations (done automatically, reported to principal)
  3. Discussion items (requires principal input)
- **Flow**: After disposition → discussion if needed → revision → potential additional review loop
- Must be documented in both Claude TheAgency.md and README TheAgency.md

### Architecture & Implementation Planning

- **A&D Phase**: Multi-agent input group provides technical approach before writing
- **Review Dimensions**: Ease of implementation, maintainability, evolution capability, performance, security, testability
- **Implementation Planning**: Multi-agent group creates "plan seed" → combined with PBR + A&D → plan mode → master plan
- **Phase Execution**: Master plan broken into phases → individual phase plans → autonomous MAR (no principal unless flagged)
- **Quality Gates**: QAR + QG + QGR at end of each iteration, phase-level quality gates unlock next stage

### Commit & Integration Workflow

- **Commit Triggers**: End of iteration or phase completion
- **Captain Role**: Receives dispatch → merges commit → syncs everyone
- **PR Process**: End of phase → Captain builds PR → pre-print quality gate → push to origin → trigger merge
- **Captain Loop**: Regular fetch from origin master, scan dispatches, handle commits
- **Batch Processing**: Multiple commits processed together before pushing to work trees

### Technical Integration Requirements

- **Platform Support**: Claude Code CLI (primary), Claude Desktop code tab, remote connections (mobile devices, laptops)
- **Human Notification**: MDPal tray for sending messages to humans outside of stream
- **Artifact Storage**: Need to define storage system and naming conventions for all work artifacts
- **Quality Gate Enforcement**: Must execute before commits allowed, blocking mechanism required
- **Tooling Needs**: Quick/easy writing tools for artifacts and signing process
