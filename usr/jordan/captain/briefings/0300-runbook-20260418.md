## Workstream B — Shebang investigation + resolution

**Purpose:** Resolve whether framework Python tools should declare a specific interpreter (`python3.13`) or a flexible floor (`python3` + `sys.version_info` guard). Must complete before Workstream A edits any file.

### Output

`usr/jordan/captain/briefings/python-shebang-investigation-20260418.md` — structured briefing:

1. **Current convention:** `#!/usr/bin/env python3.12` (D44 decision, now stale) — hard-codes exact binary name, requires `python@X.Y` to be brew-installed with that exact name on PATH.
2. **Options:**
   - **B1:** `#!/usr/bin/env python3.13` — exact binary. Mechanical enforcement. Adopter friction if they have 3.13 but not named `python3.13` (some package managers install as `python3` only).
   - **B2:** `#!/usr/bin/env python3` + preamble guard:
     ```python
     import sys
     if sys.version_info < (3, 13):
         sys.exit("Python 3.13+ required (got {}.{})".format(*sys.version_info[:2]))
     ```
     Flexible — any `python3` that meets the floor works. Adopter-friendly.
   - **B3:** Hybrid — `#!/usr/bin/env python3` shebang + guard, PLUS `agency-health` check that warns if `python3` resolves to < 3.13.
3. **Tradeoffs:** mechanical enforcement vs adopter ergonomics; shebang discovery vs runtime fail-fast; portability across macOS (brew, Apple stock) / Linux (apt, deadsnakes, pyenv).
4. **Recommendation:** **B3** (hybrid) — `python3` shebang gives adopters the "just works" experience; the guard gives runtime fail-fast; `agency-health` gives install-time visibility. Workstream A implements B3.
5. **Fallback:** If B3 exceeds scope for a same-night PR, default to B2. Do NOT default to B1 (same fragility as what we hit tonight).
