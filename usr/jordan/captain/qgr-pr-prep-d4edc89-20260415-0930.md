# QGR — D41-R2 pr-prep

## Scope

Add `git-captain sync-main` subcommand to fast-forward local main to origin/main.
Required by `/post-merge` after a PR is merged on GitHub. Without it, captain
(on main) has no safe tool for the ff step — all raw git is blocked by hookify.

**Files changed:**
- `agency/tools/git-captain` (+33 lines: new subcommand, usage line, dispatch)
- `tests/tools/git-captain-sync-main.bats` (new, 8 tests)

## Issues Found and Fixed

| ID | Severity | Source         | Description                                                               | Fix                                                                 |
|----|----------|----------------|---------------------------------------------------------------------------|---------------------------------------------------------------------|
| 1  | bug      | reviewer-code  | Detached HEAD → misleading "Must be on main. Currently on: " (empty)      | Explicit `[[ -z "$current_branch" ]]` check + actionable message    |
| 2  | concern  | reviewer-code  | Extra args silently ignored (e.g. `git-captain sync-main foo`)            | `[[ $# -gt 0 ]] && die "Usage: git-captain sync-main (takes no args)"` |
| 3  | coverage | reviewer-test  | Zero BATS coverage — git-captain.bats doesn't exist for any subcommand    | New tests/tools/git-captain-sync-main.bats (8 tests, sync-main scope only) |

## What Was Found and Fixed (narrative)

Three reviewer agents examined the 12-line addition to git-captain and the
zero-coverage baseline. reviewer-code flagged two correctness issues:
detached HEAD produces a misleading error message, and the subcommand
silently accepts trailing arguments. Both fixed with explicit guards and
actionable error text. reviewer-design confirmed the addition matches
sibling subcommand conventions (color codes, die() pattern, detect_main_branch
usage, dirty-tree check style). reviewer-test identified the absence of any
BATS coverage for git-captain — a pre-existing gap. Filed a BATS suite of 8
tests scoped to sync-main (happy path, not-on-main, dirty-tree, detached-HEAD,
missing origin/main, no-op FF, diverged-history, extra-args). All pass. The
broader git-captain coverage gap is flagged separately (#128) for a dedicated
testing iteration — not blocking this release.
