# Feedback Registry — Anthropic / Claude Code

Tracks feedback items filed to Anthropic about Claude Code, SDK, and related products. Each entry records the feedback ID, GitHub issue URL, date, topic, and current status.

## Format

One markdown file per feedback item, plus `registry.md` as the index.

Naming: `feedback-{YYYYMMDD}-{slug}.md`

## Index

See `registry.md` for the sorted list.

## Why track

1. **Follow-up** — I know what's outstanding and can ping/update when state changes
2. **Pattern visibility** — recurring themes across filed items surface product-level observations worth writing up
3. **Audit trail** — if filing mechanism (`/feedback`) changes, we have our own record
4. **Reporter reference** — can cite prior feedback when filing related items
