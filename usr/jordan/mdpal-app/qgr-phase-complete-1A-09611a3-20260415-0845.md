## Cross-iteration consistency (own review)

1. **Error surfacing is uniform.** `document.lastError` is the one-line error bus for mutations. The edit-flow version-conflict intentionally bypasses it to drive a distinct Overwrite/Discard/Keep-editing alert (1A.4) — this is a design choice, documented in that iteration's QGR. Other errors from edit also route to `document.lastError` so the 1A.3 alert handles them uniformly. ✓

2. **Sheet error-return pattern is consistent.** AddComment / FlagEditor / ResolveComment sheets all use the `Bool`-returning `onSubmit` established in 1A.2. Sheets stay open on failure, drafts preserved. ✓

3. **Mocks grew additively, not by duplication.** `ToggleTrackingService` (1A.2) gained `FailingToggleService` as a wrapper (1A.3). Real mock (`MockCLIService`) was reused for edit tests (1A.4) because its version-hash enforcement was already correct. ✓

4. **Helpers are appropriately scoped.** `EditConflict` (1A.4) is a nested view-scoped struct; `SelectionContext` + `ClipboardReader` (1A.5) are public file-scoped types so they can be tested from the test target. Both have single responsibilities. ✓

5. **Optional-DocumentModel init preserved for previews/tests.** The read-only `SectionReaderView.init(section:comments:flag:)` still compiles; interactive features hide cleanly when `document == nil`. ✓
